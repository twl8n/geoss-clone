Downloading GEOSS

     

Why download?
If you wish to install GEOSS on your own server, you'll want to download the latest version. The download files are gzipped tar archives (we have plans for a RPM binary installation) Extensive installation instructions are inside the archive. The most recent version is the most stable. The installation typically takes a day, although we have performed it from scratch in a couple of hours. The developers are always happy to field background questions or technical questions about the product. Email: geoss-help@virginia.edu

GEOSS is known to run on RedHat Linux, Fedora, and Apple OSX systems. PostgreSQL (aka Postgres), Apache, R, and several Perl modules are required. GEOSS is written in Perl. End users interact with the system only via an easy to use Web interface. Administration mostly happens at the Linux command line (also documented, and easy). The Microarray Center workflow is managed via a web interface.

Other Packages
To complete the installation, you'll need some ancillary software. Here we have a synopsis. Complete instructions are in the installation guide in the archive.

Perl Modules

AppConfig-1.52

IO-stringy-2.108

Spreadsheet-ParseExcel-0.2602

OLE-Storage_Lite-0.10
We are using version 0.10, but the current version at CPAN is 0.11. I suggest that you download the current version.

DBD-Pg-1.13
We are using version 1.13, and again I suggest that you download the current version. Older versions might not be compatible with the current version of Postgres.

DBI-1.29
We are using 1.29 which I think is included as part of Perl 5.8. If you need DBI, I suggest that you download the latest version.

Compress-Zlib-1.21
I think the minimum required by PDF::API2 is 1.03. We are using 1.21

PDF-API2-0.3b49
This is a PDF creation API used by the tree draw Analysis Tree module. It requires Compress-Zlib (above).

The R statistical package R-1.6.1-2
http://cran.r-project.org/bin/linux/redhat/8.x/i386/
http://cran.r-project.org/
We started out running RedHat 7.x and 8.x with R 1.6.1-2. We have upgraded to Fedora. The CRAN developers have a newer version out, which we suggest you use. The link above is to the directory containing the binary rpm for RH 8. Versions of R are available for Linux, Unix and Windows systems.

Apache
The Apache web server is included with Red Hat. If you are running Red Hat, I suggest that you use the Red Hat Apache distribution. If you need to download a copy of Apache, I suggest that you use Apache 2. There are very few (or no differences) between configuring Apache 1.3.x and Apache 2.x for GEOSS.
The download page: http://httpd.apache.org/download.cgi
It may be easiest to download the source package. Use the link below. For Linux, download the highest numbered .tar.gz file:
http://www.apache.org/dist/httpd/

PostgreSQL
Postgres also comes with Red Hat 8. If you need it, you can try the PostgreSQL site:
http://www.postgresql.org/
If you must download, you'll need most of the files from the ftp site below. I've given an example URL here to the binary installation for PostgreSQL 7.3.1 for Red Hat 8.
ftp://ftp10.us.postgresql.org/pub/postgresql/binary/v7.3.1/RPMS/redhat-8.0/

multtest 1.0 (optional, only used by Westfall & Young)
This extra R package contains some additional testing procedures, currently only used by the Westfall and Young analysis.
http://bioconductor.org/repository/release1.1/package/html/multtest.html
Installation is easy. su to root, then use the following command (all caps required!)
R -CMD INSTALL multtest_1.0.tar.gz
If you want to remove it try this:
R -CMD REMOVE multtest
If you can't remember a package name, just ls /usr/lib/R/library.
After installation, documentation (text, html, and latex, respectively) is in:
/usr/lib/R/library/multtest/help/
/usr/lib/R/library/multtest/html/
/usr/lib/R/library/multtest/latex/

Download GEOSS via CVS
We recommend that you download the tar.gz archive. It contains the complete sources and documentation.

You may use the following CVS commands to download the most recent stable version of GEOSS, or the most recent version. Each of the following command is a long line.

  1. Change directory to your desired location for the GEOSS installation files. This will not be the final installation directory.
  2. Enter only one of these commands at the shell prompt. GEOSS is most compatible with Red Hat 7 or newer systems.
  3. We recommend that you use the stable release. The feature set is nearly complete, with the exception of the Analysis Trees.

 
     


The current stable release (recommended since it has all the interesting features, and is stable):

cvs -d:pserver:anonymous@cvs.va-geoss.sourceforge.net:/cvsroot/va-geoss co -r Rel-1_6_11 va-geoss
