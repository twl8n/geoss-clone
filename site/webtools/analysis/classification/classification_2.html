Description
The GEOSS module 'Classification' provides optimal sets of genes that separate samples into two classes. The output from this module is formatted in a table structure for easier access and readability. Classification version 2 uses R 2.4.0.


Statistical Methods
Classification can be performed by LDA, QDA, SVM, or logistic regression based on MiPP. For explanation, look at:

    1. Soukup M, Cho H, and Lee JK (2005). Robust classification modeling on microarray data using misclassification penalized posterior, Bioinformatics, 21 (Sppl), i423-i430.
    2. Soukup M and Lee JK (2004). Developing optimal prediction models for cancer classification using gene expression data, Journal of Bioinformatics and Computational Biology, 1(4) 681-694.


Input files
Classification expects input in this form:

  • A file containing chip data. The file should have row and column labels.

Analyses that create input in this format include:
  • Quality Control
  • Filter
  • Multi-Condition Filter

Output files
Classification creates the following output:

  • A output file containing optimal sets of genes and classification error measurements
Analyses that accept filter output as input include:
  • None

 

Parameters:

  • 'Number of splits' is the number of independent splits into train and test sets. For each split, a different model may be selected.
  • 'Number of sequential runs' is the number of sequential runs after eliminating the genes selected in the previous runs. This may discover masked genes.
  • 'Cutoff point for model selection' is the cutoff point of 5% sMiPP to select optimal models for each split.
  • 'Define new conditions if greater than 2' allows one to define new conditions, particularly make multiple conditions into two conditions. Type in 1 or 2 to define new conditions and leave boxes 'blank' to exclude unused conditions. If all boxes are blank, only the first two conditions are used.
  • If an independent test set for each condition is available, click 'Yes' in 'Are train and test sets distinct?' and specify which ones are train or test sets in 'Specify if train and test sets are distinct'.

 

Note:

  • Note that multiple classes are not accepted at present, so if there are more than two classes, you should combine multiple conditions into two conditions.
  • When an independent test set for each condition is available, one train set and one test set must be specified for each of new conditions 1 and 2.
