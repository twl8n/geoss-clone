Description
The cluster analysis performs hierarchical clustering analysis for microarray data (it is recommended to apply this module for an array data set with less than 2,000 genes for optimal performance). One can choose different combinations of one of the distance metrics (e.g., Euclidean, Pearson correlation, Spearman correlation) and one of clustering algorithms (e.g., "single", "complete", "average").

Cluster then draws the hierarchical clustering tree and corresponding image maps (cluster-image map; CIM) from the output of a hierarchical clustering algorithm. This will generate a high-density pdf graphic, which is zoomable by utilizing Adobe Acrobat reader. The gene names and annotations at the end of the tree branches are hyperlinked to genome web sites.
Cluster version 2 is run using R 2.1. Cluster version 1 is run using R 1.9.


Input files
Cluster expects input in this form:

  • A file containing an analysis criteria. The file should have row and column labels. Note that we won't process more than 7000 rows.

Analyses that create input in this format include:
  • Filter
  • Differential Discovery (likely too many rows - run through filter first)

 

Output files
Cluster creates the following output:

  • A file containing data necessary to draw a graph
  • A file containing user data
  • A file containing stat data
  • A pdf file showing a cluster tree
Analyses that accept cluster output as input include:
  • None

 

Parameters:

  • Graphical output filename - name of the file containing the tree
  • Zoom factor - zoom increases "magnification" of the tree but not the colorized expression map
  • Need of log 2 transformation - log2-transformation is taken typically and the previous modules provide raw data, so this should be set to 'True', unless log2-transformation is taken elsewhere.
  • Euclidean distance measure - the distance/dissimilarity metric.
    "Euclidean" - Euclidean distance
    "Correlation" - Pearson correlation
    "Spearman correlation" - Spearman rank correlation
  • Clustering algorithm
    "average" - average distance
    "complete" - maximum distance or farthest neighbor
    "single" - minimum distance or nearest neighbor
  • Log filename - name of the file containing log and error messages
  • Statistical graph, user, Tstat output filenames are obsolete
