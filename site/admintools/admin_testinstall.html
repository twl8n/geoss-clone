Build Parameters:
  • GEOSS_DIR={geoss_dir}
  • HTML_DIR={html_dir}
  • WEB_DIR={web_dir}
  • USER_DATA_DIR={user_data_dir}
  • BIN_DIR={bin_dir}
  • LIB_DIR={lib_dir}
  • GEOSS_DBMS={geoss_dbms}
  • DB_NAME={db_name}
  • GEOSS_HOST={geoss_host}
  • GEOSS_PORT={geoss_port}
  • GEOSS_SU_USER={geoss_su_user}
  • PERL_LOCATION={perl_location}
  • COOKIE_NAME={cookie_name}
  • VERSION={version}
These options were set at build time in build-options.mk. If these options are changed, the system will need to be rebuilt for changes to take effect. Changes to these options make require simultaneous changes to the database and web server configuration.

Installed Versions for Required Software:
  • Web Server: {web_server}
  • Database: {database}
  • R Programming Language:
    • Version 1.9 (optional): {rlang}
    • Version 2.1: {rlang2.1}
Above applications are required for complete functionality. GEOSS will run without R, but most analyses will not function.

Directory Access:
  • Read Access User Data Dir ({USER_DATA_DIR}): {r_user_data_dir}
  • Write Access User Data Dir ({USER_DATA_DIR}): {w_user_data_dir}
  • Read Access to Chip Data Dir ({CHIP_DATA_PATH}): {r_chip_data_dir}
For complete GEOSS functionality, it is necessary for the user running the GEOSS scripts to have the designated permissions on the above directories. If any of the directories have a permission failure, it should be investigated immediately.
 

Database Security:

  • Permissions for {db_config_file}: {db_config_pass}
The database configuration file should have read/write permissions for the {web_user} user only. It is a security issue if other users can access this file. Permissions for all other users should be revoked.

Layouts Loaded: {installed_layouts}
Array Center Staff will only be able to load data for chips that have a layout loaded.

Analyses Loaded: {installed_analyses}
Analysis tree functionality will only be available if one or more analyses are loaded.
