3.0 FAQ

  • 3.1 Analysis
    • 3.1.1 My analysis didn't generate result files. What happened?
      If you have waited the appropriate amount of time and there are no output files, then one of the analysis modules has crashed. The crash could be caused by any number of things. If you are using the filter module, make sure you have selected at least two conditions. You can also check the contents of log files in your file repository as they may help isolate the problem. Enlist your administrator's help in determining why the crash occurred.
    • 3.1.2 Why can't I view the results of treedraw analysis?
      If you are trying to view the results of the treedraw analysis using Acrobat Reader and Reader crashes, you may have run treedraw with too many input rows. You may either view the output pdf using a different pdf viewer, like xpdf, or you should try using the filter module to restrict the number of rows treedraw takes as input. There is more information about this bug in the treedraw analysis help pages.
  • 3.2 Orders
    • 3.2.1 What are the different order states and what do they mean?
      Orders can have the following states:
      • Incomplete: The order has been created. Configuration may have begun, but it is not yet complete due to an insufficient number of samples or hyrbidizations.
      • Complete: Order configuration meets minimum requirements for a complete order (at least one sample and at least one sample per hybridization).
      • Submitted: Once an order is complete, it may be submitted. A submitted order is locked and you may no longer edit the order. When you submit an order, the notification process to let others know the order is ready is initiated.
      • Approved: If the order is associated with an organization, it can may require approval. Once it is submitted, it will be sent to for approval. If no approval is required, it is automatically approved and sent to the array center for processing.
      • Finished: An order is finished when all chip data has been loaded for the order.
